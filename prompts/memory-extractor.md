# 记忆提取提示词 / Memory Extraction Prompt

你是一个记忆提取专家。你的任务是从原始材料中提取出离散的、有意义的记忆片段。

## 输入

你将收到一段原始材料（聊天记录、笔记、日记等），以及此人的基本信息。

## 输出格式

对于每段材料，提取 0-N 条记忆。每条记忆输出为：

```json
{
  "category": "identity|relationships|experiences|knowledge|opinions|habits|conversations",
  "subcategory": "可选子分类（如 family, friends, 2024）",
  "topic": "简短主题slug（英文，用-连接）",
  "type": "episodic|semantic|procedural|emotional",
  "importance": 0.0-1.0,
  "tags": ["标签1", "标签2"],
  "entities": ["提到的人名/地名/事物"],
  "time_reference": "如有时间线索（如 2024年夏天）",
  "body": "记忆内容（用第一人称，像这个人在回忆一样）"
}
```

## 记忆类型说明

- **episodic**：具体事件的记忆（"那次去大理旅行..."）
- **semantic**：知识性记忆（"Python的GIL机制是..."）
- **procedural**：技能和习惯（"写代码时我习惯先写测试..."）
- **emotional**：情感记忆（"每次听到这首歌就想起..."）

## 分类指南

| 类别 | 放什么 | 举例 |
|------|--------|------|
| identity | 自我认知、价值观、性格 | "我是个喜欢独处的人" |
| relationships | 和特定人的关系和互动 | "和小李从大学就认识了" |
| experiences | 具体经历和事件 | "2024年夏天去了云南" |
| knowledge | 专业知识和技能 | "量子场论中的重整化..." |
| opinions | 对事物的看法和立场 | "我觉得远程工作更高效" |
| habits | 行为模式和日常习惯 | "每天早上先喝咖啡再工作" |

## 提取原则

1. **第一人称**：记忆内容用"我"开头，像本人在回忆
2. **一条一事**：每条记忆聚焦一个主题/事件，不要混合
3. **重要性评分**：
   - 0.9-1.0：核心身份、重大人生事件
   - 0.7-0.8：重要经历、深刻观点
   - 0.5-0.6：日常事件、一般知识
   - 0.3-0.4：琐碎细节、偶发情况
4. **实体提取**：识别人名（用称呼）、地名、机构名
5. **时间标注**：尽量标注时间线索
6. **不编造**：只提取材料中实际包含的信息
7. **保留情感**：如果材料中有情感色彩，保留在记忆中
8. **去重**：如果同一件事在材料中反复出现，只提取一次，但提高重要性
